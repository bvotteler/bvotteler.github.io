Hugo Calendly Shortcode

I’ve built hugo-calendly-shortcode to make it easier to add Calendly to your Hugo pages.
Simply add a shortcode, pick a style, and add your calendar/event name. That’s it.

Right now, the basic, free & open-source version is available here. I am also building out a more advanced version which I will offer at a price far less than the time it would take a developer to build their own.

Who is / isn’t this for?

Realistically, if you only want to embed one Calendly calendar/event on one page and don’t plan on touching it ever again, you’re most likely better off by simply adding the HTML/Javascript snippet directly into your page.
Seriously, Calendly makes it super easy.

However, if you want to embed Calendly on various pages, maybe with different events and/or for different people, maybe even update those occasionally, then it becomes cumbersome to keep all those embeds aligned.
This is what I built the shortcode for; to make it somewhat easier to have consistent embeds.

Also, the shortcode is fairly easy to use: Just add a calendar name and you’re set. Maybe tweak the style if you do not want the default link popup and/or tweak the link text.

Tell me about the advanced version

I am still in the process of building it out. Particularly adding features that take advantage of Calendly Pro/Plus subscriptions that allow you to tweak styles of the booking pages. But there are also other pieces that will make working with it even easier.

The additional features will enable you to:

  • Apply CSS styles to customize the look and feel for Calendly
  • Add UTM parameters to track your embedded Calendly forms
  • Pull parameter sets from different sources:
    • Shortcode parameters (same as with this version),
    • Hugo page front-matter,
    • Hugo site configuration parameters,
    • Hugo data files, or
    • A mix of all of the above.

So, basically I make the shortcode smarter by looking in multiple locations to find and apply defaults. Meaning, it can derive the CSS styles from Hugo’s global configuration, look at data files for embed styles per calendar, or front-matter on a particular page even.

In a nutshell, the closer you get to the actual shortcode on a page itself, the higher the priority of a setting. Meaning you can override defaults with increasing granularity if you wish to do so.
